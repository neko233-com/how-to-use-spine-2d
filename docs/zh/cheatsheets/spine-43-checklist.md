# Spine 4.3 工程检查表

每次完成一个工程，都按这页检查一遍。

## 文件

- `.spine` 源文件已保存。
- `images` 路径使用相对工程目录的稳定路径。
- PNG 文件名小写、无空格、无中文标点。
- 不再使用的图片已从 Images 面板清理。
- 导出文件放到 `spine-demos/06-export-runtime-pack` 或你的引擎资源目录。

## 骨架

- `root`、`hip`、`chest` 分工明确。
- 左右命名统一 `_l`、`_r`。
- 关节原点正确。
- slot 名和 attachment 名能一眼看懂。
- draw order 在正面、侧身、攻击姿势都合理。

## 动画

- 循环动画首尾一致。
- 非循环动画能自然回到 idle 或进入下一个状态。
- 关键事件命名稳定：`hit`、`footstep_l`、`footstep_r`、`land`。
- 删除了无意间记录的多余 timeline。
- 曲线经过检查，不是全线性或全自动柔和。

## Mesh

- 顶点数量和效果匹配，没有过度细分。
- 变形时没有破洞、尖刺、明显拉伸。
- 权重影响骨骼数量可控。
- deform key 没有覆盖无关动画。

## 导出

- JSON 导出用于调试，`.skel` 导出用于正式测试。
- Atlas padding、bleed、max size 已检查。
- 纹理数量符合目标平台预算。
- 引擎运行时版本与 Spine 编辑器 4.3 匹配。
- 事件在运行时能被监听到。
