# Spine 4.3 实战技巧速查

这页专门记录那些教程里容易一笔带过、但实战里会反复踩坑的细节。

## 素材阶段

- 每个零件都要留透明边距，旋转时才不会被裁边。
- 左右肢体不要只镜像命名，最好真的保存 `_l` 和 `_r`，后期换装更清楚。
- 武器、特效、表情做成可换 attachment，不要画死在身体图里。
- 角色朝向要统一。一个三分之二侧面角色混入正面手臂，会让绑定很难自然。
- 大块先动起来，再补小零件。新手素材越细，越容易停在整理文件阶段。

## 绑定阶段

- 骨骼长度不一定等于图片长度，但关节位置必须准确。
- slot 决定绘制层，bone 决定运动层，不要混淆。
- draw order 能解决前后遮挡，不要用移动 z 轴的思维做 2D。
- root 控整体位移，hip 控身体重心，chest 控上半身呼吸和扭转。
- 换装优先用 skin 和 attachment，不要复制整套骨架。

## Mesh 与权重

- 硬物体不用 mesh。
- 顶点沿变形方向布置，布料下缘可以多点，固定上缘少点。
- 权重边界要平滑，但不是每个顶点都需要多个骨骼。
- 先用极少顶点测试动作，再决定是否加细节。
- deform key 很强，但会增加维护成本；能用骨骼解决的先用骨骼。

## 动画阶段

- 循环动画第一帧和最后一帧必须一致。
- 先 block 关键姿势，再调曲线。
- 动作节奏优先于画面细节。攻击要有预备、爆发、回收。
- 运动链要有相位差：身体、头发、披风、武器不要同一帧到位。
- 曲线不是越柔越好，命中、踩地、落地经常需要短暂线性或硬切。

## 事件与运行时

- 伤害、脚步、落地、取消窗口都应该用 Spine Event 标记。
- 游戏代码监听事件，不要靠“播放到第几秒”猜。
- 导出时区分调试 JSON 和正式 `.skel`。
- Atlas padding 太小会有边缘渗色，尤其是旋转或缩放时。
- 运行时混合动画前，先确定每个动画是否有清理无关 timeline。

## 常见故障

| 症状 | 多半原因 | 快速修法 |
| --- | --- | --- |
| 角色一动就散架 | 图片绑错骨骼或层级错 | 回 Pose 模式检查 parent |
| 手脚绕奇怪圆弧 | 原点不在关节 | 修 attachment transform |
| 脚底滑 | 接触帧 IK target 还在动 | 锁接触区间 target |
| 布料折出尖角 | 三角形太细或权重突变 | 重布点并平滑权重 |
| 导入游戏发白边 | 图集 bleed/padding 不足 | 重新 pack atlas |
| 混合动画抽一下 | 两个动画控制同一骨骼冲突 | 清 timeline 或调 mix |
