# 工程 04：攻击、事件与特效同步

素材路径：`spine-demos/04-combat-events/images`

目标：做一个有打击感的 `attack_01`，并学会用事件同步伤害、音效、屏幕震动和特效。

## 1. 攻击三段式

| 阶段 | 帧 | 目的 |
| --- | --- | --- |
| 预备 | 0-6 | 蓄力，让玩家看懂角色要攻击 |
| 爆发 | 7-10 | 快速出手，制造速度对比 |
| 回收 | 11-24 | 回到可控姿态，准备接下一个动作 |

有打击感的关键不是全程快，而是“慢蓄力、快命中、稳回收”。

## 2. 关键帧

| 帧 | 动作 |
| --- | --- |
| 0 | idle 姿态 |
| 4 | 身体后压，武器后拉 |
| 7 | hip 前推，chest 反向扭转 |
| 9 | 武器到命中点，触发 `hit` |
| 12 | slash 最亮，impact 出现 |
| 18 | 武器回收 |
| 24 | 回到 idle 接近姿态 |

## 3. 事件设计

建议事件：

| 事件 | 帧 | 游戏用途 |
| --- | --- | --- |
| `attack_start` | 1 | 锁输入、开始消耗耐力 |
| `whoosh` | 7 | 播放挥砍音效 |
| `hit` | 9 | 结算伤害、生成 hitbox |
| `impact` | 10 | 播放命中特效、震屏 |
| `recover` | 18 | 允许取消到下一个动作 |

事件应该表达“游戏逻辑时间点”，不要只表达“画面很好看”。

## 4. 特效 attachment

`slash_01` 到 `slash_08` 可以放在同一个 slot 里换 attachment：

1. 创建 `slot_slash`。
2. 把所有 slash 图片放入这个 slot。
3. 默认 attachment 设为空。
4. 在 7-12 帧切换 slash 图片。
5. 12 帧后清空 attachment。

impact 和 dust 也同理。这样比创建大量临时骨骼更清晰。

## 5. 曲线技巧

- 预备阶段用 Ease In，让动作越来越紧。
- 命中前一两帧可以 Linear，增强速度感。
- 回收阶段用 Ease Out，避免机械复位。
- 武器和身体不要完全同一帧到位，身体领先或滞后 1-2 帧更有力量。

## 6. 导出验证

导出 JSON 后检查：

- `events` 里能看到 `hit`。
- `animations.attack_01` 里有事件时间轴。
- 图集包含 slash 和 impact 图片。
- 在运行时事件监听里能收到 `hit`，而不是靠动画播放时间猜。
