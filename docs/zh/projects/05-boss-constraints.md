# 工程 05：大型角色、约束与性能预算

素材路径：`spine-demos/05-boss-constraints/images`

目标：做一个由石块、核心和环绕特效组成的大型 Boss。重点不是复杂画面，而是学会用约束减少手动画关键帧。

## 1. 结构策略

大型角色不要把所有东西挂在一条普通骨骼链上。建议分组：

```text
root
├── body_root
│   ├── core
│   ├── head
│   ├── arm_l
│   └── arm_r
├── orbit_root
│   ├── orbit_01 ... orbit_12
└── fx_root
```

身体、环绕物、特效分开控制，后期混合动画会容易很多。

## 2. Transform Constraint

适合场景：

- 核心发光时，周围石块轻微缩放。
- Boss 转身时，肩膀和头有跟随但不完全同步。
- 环绕物跟着 `orbit_root` 旋转，但保留自己的偏移。

做法：

1. 创建控制骨 `ctrl_breath`。
2. 给几个石块骨骼添加 Transform Constraint。
3. Mix 不要一上来 100%，先用 15%-35%。
4. 在 `idle_boss` 里只动画 `ctrl_breath`。

## 3. Path 或环绕动画

如果你只是需要魔法石环绕，最简单做法是让 `orbit_root` 转动，再让每个 `orbit_XX` 保持距离。高级做法才使用 path constraint。

新手先做可控版本：

| 帧 | `orbit_root` rotation |
| --- | --- |
| 0 | 0 |
| 60 | 180 |
| 120 | 360 |

设置 Linear 曲线，循环播放。

## 4. 性能预算

| 项目 | 建议上限 |
| --- | --- |
| 普通小怪骨骼 | 30-60 |
| 主角骨骼 | 60-120 |
| Boss 骨骼 | 120-220 |
| 单个 mesh 顶点 | 能少则少，先 20-60 |
| 同屏 Spine 数量 | 用目标平台真机测 |

Spine 优化不是只看骨骼数量。mesh 顶点、图集尺寸、透明重叠面积、混合动画数量都会影响性能。

## 5. Boss 动画清单

至少做这几个：

- `idle_boss`：呼吸、核心缩放、石块轻微漂浮。
- `telegraph_attack`：明显前摇，告诉玩家危险。
- `slam`：身体下压、拳头砸地、`hit` 事件。
- `stagger`：核心闪烁、石块散开。
- `death`：核心熄灭、环绕物掉落。

## 6. 实战建议

- 大型角色动作要慢，但命中瞬间仍然要快。
- 约束是为了减少重复劳动，不是为了显得高级。
- Boss 的 readable silhouette 比细节更重要。
- 复杂动画先关掉特效，只看身体节奏。
- 导出前用一张图集测试，再决定是否拆多图集。
