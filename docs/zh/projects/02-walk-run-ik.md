# 工程 02：行走、奔跑与脚部 IK

素材路径：`spine-demos/02-walk-run-ik/images`

目标：做出可循环的 `walk`、`run`，并用 IK 控制脚底接触。这个工程会让你理解“动画不是摆姿势，而是控制重心”。

## 1. 先做四个姿势

行走循环建议 30 FPS、24 帧。

| 帧 | 姿势 | 重心 |
| --- | --- | --- |
| 0 | 左脚接触，右脚后蹬 | 高 |
| 6 | 右脚经过身体下方 | 低 |
| 12 | 右脚接触，左脚后蹬 | 高 |
| 18 | 左脚经过身体下方 | 低 |
| 24 | 回到第 0 帧 | 高 |

不要一开始就调曲线。先让四个关键姿势成立，再补中间帧。

## 2. IK 设置

1. 给每条腿创建两段骨骼：`thigh` + `calf`。
2. 在脚踝附近创建目标骨骼：`foot_ik_l`、`foot_ik_r`。
3. 给腿添加 IK constraint，target 指向对应 `foot_ik`。
4. 在动画里移动 IK target，而不是直接旋转每段腿骨。

## 3. 脚底不滑技巧

- 脚接触地面的阶段，IK target 的 x/y 不要动。
- 身体 root 可以向前，但接触脚应该像钉在地上。
- 经过姿势时脚可以抬高，落地前加一点 Ease Out。
- 每次循环都检查第 0 帧和第 24 帧的脚位置是否完全一致。

## 4. 从 walk 变 run

复制 `walk` 为 `run`，然后做三件事：

1. 时长从 24 帧压到 16 或 18 帧。
2. hip 上下幅度增加。
3. 加入腾空帧：两只脚都短暂离地。

run 不是把 walk 播快。跑步的关键是腾空和落地冲击。

## 5. 实战调参

| 问题 | 原因 | 修法 |
| --- | --- | --- |
| 脚滑 | 接触阶段 target 还在移动 | 锁住接触帧之间的 IK target |
| 膝盖反折 | IK bend direction 不对 | 切换 bend 或调整膝盖初始方向 |
| 身体像漂浮 | hip 没有上下变化 | 接触高、经过低、腾空更高 |
| 手臂像钟摆 | 曲线太线性 | 给肩膀和前臂不同相位 |
| 循环抖一下 | 首尾帧不一致 | 复制第 0 帧到最后一帧 |

## 6. 事件

给 `walk` 添加：

- `footstep_l`：左脚落地帧。
- `footstep_r`：右脚落地帧。

这些事件以后可以驱动音效、尘土、地面检测。不要把脚步声写死在游戏代码里。
