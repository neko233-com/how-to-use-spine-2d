## 怎么使用这些案例

每个案例都不是“看看就算”的条目，而是一个可交付练习：

1. 在 `spine-demos/` 下创建对应工程目录。
2. 用当前仓库的透明 PNG 或 AI 切图素材完成 `.spine` 源文件。
3. 至少导出一次 JSON、一次 `.skel`、一次 atlas。
4. 在案例旁记录你遇到的绑定、权重、事件或运行时问题。

推荐顺序：先完成一个模块的 3-4 个案例，再进入下一模块。不要从生产级案例开始硬冲，Spine 真正难的是结构习惯，不是按钮数量。
