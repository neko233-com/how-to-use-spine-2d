# 同行工程常见生产模式

这页把公开示例和常见商业项目的做法抽象成可复用模式。它不是某个工程的复制品，而是你自己做项目时的决策表。

## 模式 1：主角综合骨架

适合：玩家角色、长期维护角色。

结构：

- `root` 控全局。
- `hip` 控重心。
- `chest` 控上身。
- 手脚有 IK target。
- weapon、face、fx 有独立 slot。
- skin 只做换装，不复制动作。

关键点：主角骨架宁可前期多留几个控制位，也不要后期为了一个武器动作重构全身。

## 模式 2：小怪量产骨架

适合：多个敌人共享动作。

结构：

- 骨骼少于主角。
- slot 稳定，attachment 差异化。
- 动作以 idle、walk、attack、hurt、death 为核心。
- 颜色、武器、头部通过 skin 区分。

关键点：小怪的价值在复用，不在每个怪都做一套精致 rig。

## 模式 3：Boss 分组骨架

适合：大型怪、机械体、召唤物。

结构：

- body、arms、fx、orbit 分组。
- 大块硬件独立骨骼。
- transform constraint 控群组呼吸。
- 事件用于危险提示、命中、震屏。

关键点：Boss 动作要先建立玩家可读的前摇，再加特效。

## 模式 4：UI Spine 动效

适合：头像、按钮、抽卡、奖励。

结构：

- 文件小，图集小。
- 动画短，循环明确。
- 多用 attachment sequence 和颜色曲线。
- 少用复杂 mesh。

关键点：UI 动效要快读、干净、可中断，不能像角色动画一样拖沓。

## 模式 5：AI 素材替换骨架

适合：快速迭代美术风格。

结构：

- 固定骨架和 slot。
- 新素材必须复用命名。
- 切图 manifest 保留源图边界。
- 每次替换只检查原点、比例、透明边缘。

关键点：AI 不是替代 rig，而是加速 attachment 生产。骨架规范越稳，AI 替换越值钱。
